# 小满伴学 · 全链路审视（2026-07-24，站在 1000 万目标上）

## 一、核心诊断：四个 tab 的冲突本质

今日/诊断/计划/图谱并列，是"我们有哪些功能"的思维。但用户脑子里没有功能清单，只有处境：
- 新用户：我刚点进来，这是什么？→ 要的是**被击中**
- 日常用户：今天带孩子干什么？→ 要的是**一个答案**
- 付费犹豫者：值不值 199？→ 要的是**信任**
- 续费犹豫者：还要不要再付一年？→ 要的是**看见进步**

四个 tab 的具体冲突：
1. **今日 vs 计划**：两份任务清单并存，用户不知道以哪个为准（计划是诊断生成的静态清单，今日是动态任务卡——本质是同一件事的两个半成品）
2. **诊断是一个"事件"，不是一个"目的地"**——没人每天去"诊断 tab"逛逛；诊断发生在新客激活和阶段性复测，应该是一条流程，不是一个 tab
3. **图谱是 showcase 和成就墙**，是"看"的地方，和"做"的地方（今日）并列会造成角色混乱
4. 结果：每个 tab 都在服务漏斗的多个环，每个环的体验都被稀释

## 二、漏斗思维的信息架构（IA）提案

原则：**产品只有一个家（今日），其余都是房间。**

```
【新客线】短视频 → 快诊断落地页(无 tab 框架，全屏沉浸) → 迷你报告 → 补一个断点(免费) → 完整诊断 → 体检报告 → 会员墙 → 落入今日
【日常线】打开 = 今日（唯一的家）→ 忆→练→学 → 庆祝 → 明天再来
【成长线】今日里点"看看进步" → 成长页（掌握度曲线 + 星空图谱成就墙 + 周报）
【分享线】报告/进步 → 海报生成 → 微信群 → 新客线
```

**三个 tab：**
- **今日**（家）：每天唯一要做的事。计划 tab 并入——过期的、今天的、复习的全在这，一份清单
- **成长**：孩子的进步宇宙——掌握度曲线、streak 历史、**星空图谱挪到这里**（从"工具"改名叫"知识星空"，已掌握的节点发光，是成就墙不是作业本）、周报
- **我的**：会员、档案、设置

**诊断不再是 tab**：它是两条线里的流程——新客从落地页进，老客从今日的"阶段复测"或成长页的"再测一次"进。测完落到报告，报告导回今日。

## 三、全链路逐环审视

| 漏斗环 | 服务界面 | 现状 | 要做的 |
|---|---|---|---|
| 获客 | 快诊断落地页 | 已有但裹在 tab 框架里 | 新客访问时全屏沉浸（无 tab 栏），短视频链接带参数直接进入 |
| 激活 | 快诊断→迷你报告→补一个断点 | 报告页五扇门 | 线性漏斗，每屏一个主动作（上轮讨论已定） |
| 留存 | 今日 | 与计划重复、无连续感 | 计划并入今日；streak；教练记忆话术 |
| 收入 | 会员墙（清单+补给包） | 激活码演示态 | 等支付资质；先把墙的文案和时机打磨好 |
| 续费 | 成长页 | 不存在 | 掌握度周曲线+星空成就墙+周报——**续费率 ≥50% 的战场全在这** |
| 推荐 | 分享海报 | 不存在 | 报告海报+进步海报（canvas 生成） |

# 中国 K12 知识地图（cn-k12-map）

一个「中国 K12 知识点树 + 交互式知识地图」项目：把中国 K12 阶段（小学 1 年级到高中 12 年级）的知识点组织成一张**有向无环的先修依赖图**，并用一个单页交互地图可视化。

- **每个点** = 一个微知识点（topic）
- **颜色** = 学科（15 个学科）
- **纵轴** = 年级（1 在下，12 在上）
- **连线** = 先修关系（hard = 真正的逻辑前置，soft = 学了更好）

数据 schema 与可视化思路借鉴开源项目 Marble Skill Taxonomy 的设计思想（不复制其任何内容），全部内容为中文原创撰写。

## 语文学科说明

语文不按课文篇目（篇目逐年变）而按 **能力主线** 组织：以 2022 版义教语文课标六大学习任务群为 domain（语言文字积累与梳理 / 实用性阅读与交流 / 文学阅读与创意表达 / 思辨性阅读与表达 / 整本书阅读 / 跨学科学习），外加低中年级打底的「汉语拼音与识字写字」，高中段（10-12 年级）对应高中课标任务群（整本书阅读与研讨《乡土中国》《红楼梦》、思辨性阅读与表达、文学阅读与写作等）。年级覆盖 1-12，先修链示例：拼音 → 识字 → 独立阅读；写话 → 片段习作 → 完整记叙文 → 议论文；古诗诵读 → 文言实词 → 文言翻译与鉴赏。

## 快速开始

```bash
npm install
npm run dev        # vite 静态服务器，浏览器打开提示的地址
```

交互说明：

- **滚轮**缩放，**拖拽**平移
- **点击节点**：右侧显示详情（描述、类型、年级、掌握证据、家长检验话术、课标对齐、单元概述），并高亮它的「先修链」（向上递归：先学这些）和「解锁链」（向下递归：学会后能解锁什么）
- **hover**：显示名称 tooltip
- **搜索框**：按名称 / 描述 / 单元模糊搜索，点击结果直达
- **学科 chips**：按学科显示 / 隐藏
- 默认只显示道法学科内部的先修边，勾选「显示全部先修线」可看到骨架学科的纵向进阶边
- **掌握度标记**：节点详情里「已掌握 / 还不会」自评，存 localStorage（`cnk12_mastery`），驱动顶部统计与「下一步学什么」推荐
- **错因回溯**：详情里「帮我找原因」沿完整先修链逐节点自评，自动定位最根上的缺口
- **家长检验卡**：每个完整知识点附检验话术与「怎么算答对」清单（家长身份默认展开，学生默认折叠）

## 学习闭环与会员功能

免费功能：地图浏览、自评标记、先修链、搜索、学习推荐全部可用。以下功能为会员专享（localStorage `cnk12_member`）：

- **测一测（节点诊断测）**：道法 143 个知识点各配 3 道题（`data/quiz.json`，429 题，单选 + 判断）。答完立即给对错与解析；3/3 自动标记「已掌握」，否则标记「还不会」并给出错题回顾，可一键接「帮我找原因」或「看学习资源」；「再测一次」题目与选项重新打乱即为复测
- **学习资源区**：每个道法知识点配套课程 / 视频 / 练习 / 方法文章（`data/resources.json`），外链新标签打开，底部「学完了，再测一次」回到答题，形成「诊断测 → 找原因 → 学资源 → 复测」闭环
- **快速诊断**：工具栏入口，选学科（目前仅道法有题库，其余置灰「题库建设中」）→ 选年级（7/8/9）→ 跨节点随机抽 10 题逐题作答 → 结果按知识点分组（答对绿色 / 答错橙红），点击跳转地图节点详情，「把结果标记到战况图」一键批量写入掌握度
- **会员门**：未开通时点上述入口弹出会员引导卡（权益列表 + 激活码输入）。v1 占位激活码 `VIP2026`（界面注明"内测激活码，正式支付即将接入"），开通后工具栏显示「会员 ✓」，「会员」按钮可查看状态与权益

## 数据校验

```bash
npm run validate     # 校验：id 唯一、无悬空引用、无环（拓扑排序）、字段合法
npm run build-data   # 编辑 data/topics.json / dependencies.json 后，重新生成 manifest.json
```

## 目录结构

```
cn-k12-map/
  package.json
  index.html                     # 交互式知识地图（纯前端 Canvas 单页）
  data/
    topics.json                  # 微知识点（图节点）
    dependencies.json            # 先修依赖边（DAG）
    curriculum-standards.json    # 课标条目
    clusters.json                # 每个学科×领域的家长友好概述
    quiz.json                    # 节点诊断题库（429 题，道法 143 节点 × 3 题）
    resources.json               # 节点配套学习资源（道法 143 节点）
    manifest.json                # 数量统计（由 build-data 生成）
  scripts/
    build-data.mjs               # 重新生成 manifest.json
    validate.mjs                 # 校验脚本
```

## 数据 schema

### topics.json

```json
{
  "topics": [{
    "id": "cn_df_xf_01",
    "type": "CONCEPTUAL | PROCEDURAL | REPRESENTATIONAL | LANGUAGE | META",
    "subject": "道德与法治",
    "domain": "坚持宪法至上",
    "name": "宪法是国家的根本法",
    "description": "1-2 句白话解释",
    "gradeRangeStart": 8,
    "gradeRangeEnd": 8,
    "status": "complete | skeleton",
    "evidence": ["能说出……", "能举例……"],
    "assessmentPrompt": "如果孩子……，能……吗？",
    "standards": ["yw-2022:道德与法治.法治教育.第四学段"]
  }]
}
```

### dependencies.json

```json
{
  "dependencies": [{
    "topicId": "cn_df_xf_01",
    "prerequisiteId": "cn_df_fz_02",
    "strength": "hard | soft",
    "reason": "一句话说明为什么先学它（必须具体）"
  }]
}
```

必须是**有向无环图（DAG）**，`validate.mjs` 会做拓扑排序检查。

## 内容原则

- 道法内容按 **2024 版统编《道德与法治》** 六册 24 个单元组织（九下为课标推导、待新教材出版后核对），贴合教材与中考方向，不编造概念
- 语文内容按 2022 版义教课标六大学习任务群 + 高中课标任务群组织，以能力为主线而非课文篇目
- description 说人话（家长能看懂），evidence 是可观察的掌握标准
- 依赖边的 reason 必须写具体逻辑（为什么它是前置），禁止「因为需要先学」式废话
- 骨架学科节点只搭结构，界面中明确标注「骨架占位，内容待填充」
- 道法/语文数据由 `scripts/gen_df*.py`、`scripts/gen_yw*.py` + `scripts/gen_build.py` 生成，修改内容请改生成脚本后重跑，再执行 `npm run validate && npm run build-data`

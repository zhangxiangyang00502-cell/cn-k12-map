# 小满伴学 · 千万收入路径推演（2026-07-24）

## 一、1000 万的数学

年收入 1000 万 ÷ 客单价（早鸟 199 / 正价 399，混合取 **¥299**）= **约 3.3 万个年付费家长**。

倒推漏斗：
- 免费诊断 → 付费转化按 5%（工具类乐观值）→ 需要 **67 万免费诊断用户/年**（每月 5.6 万）
- 获客不买量（买不起也违背定位）→ 只能靠**短视频矩阵 + 家长群分享**
- 短视频：播放→点击诊断 1-3% → 需要 **3000-7000 万播放/年** → 每天 10-20 条视频持续分发
- **续费率 ≥50%** 是生死线：第二年 1000 万里必须有一半来自续费，否则获客压力翻倍

## 二、数学反推出的三句话

1. **获客 = 你已有的引擎。** leap_leads 就是干短视频内容生产的——中考道法/语文的"家长焦虑内容"（"这道中考题 70% 孩子答错"）→ 评论区/主页挂快诊断链接。我们的产品和 leap_leads 是同一台机器的两半：它负责流量，小满负责变现。
2. **转化 = 快诊断落地页 + 体检报告。** 短视频来的用户 3 秒内必须开始答题，答完 2 分钟内必须看到"原来如此"的报告，报告页底部才是 ¥199。这条链路每一个跳转都在漏人，要按落地页标准打磨。
3. **续费 = 教练 + 内容厚度 + 周报。** 家长续费的理由只有一个："这东西真的在带我孩子进步"。所以 AI 教练（后端）和掌握度曲线不是体验优化，是收入结构。

## 三、产品架构差距（现在的静态 H5 撑不起 1000 万）

必须有轻后端四件套（按依赖排序）：
1. **账号体系**——微信授权登录（家长都在微信里），掌握度/打卡数据云端同步（换手机不丢，这是付费的前提信任）
2. **微信支付**——个体工商户即可开商户号，年费类虚拟商品需要 ICP 备案的域名
3. **LLM 代理**——教练对话、说一说升级、拍整卷，API key 不能暴露在前端
4. **数据看板**——我们自己要看漏斗数据（诊断完成率、分享率、付费率、留存），没有数据就是在黑飞

技术路线建议：微信小程序（家长免下载、分享卡片原生支持、支付无缝）或 H5+微信 JSSDK。静态前端可以继续用，后端用一个轻服务（CloudBase/LeanCloud/自建 Node 均可）。

**需要你办的事（阻塞项）：**
- 注册个体工商户或公司（小满即安）
- 域名 + ICP 备案（约 2-3 周）
- 微信商户号 + 微信开放平台账号

## 四、UI/体验在 1000 万视角下的角色

漏斗每一环对应一个体验指标，UI 是为一转化率服务的：
- 播放→答题：**快诊断落地页**，零选择直接出题（目标 ≥30% 开始答题）
- 答题→报告：答题过程零摩擦（目标 ≥70% 完成率）
- 报告→付费：报告页制造"原来如此 + 还差一步"（目标 ≥5%）
- 付费→留存：打卡特效、streak、周报（目标次日留存 ≥40%、7 日 ≥25%）
- 留存→分享：晒点海报（目标分享率 ≥8%，K 因子 >0.3 就能摊薄获客）

## 五、分阶段里程碑

- **M1（现在-4 周）**：体验攻坚（P0 打卡特效+快诊断、P1 海报+streak+报告页）。验收指标：次日留存 ≥30%、分享率 ≥8%
- **M2（4-8 周）**：资质办妥 + 支付上线 + 首批 10 条短视频引流测试。验收：播放→诊断 ≥2%、**50 个真实付费家长**（第一笔 1 万元，PMF 真验证）
- **M3（2-4 月）**：轻后端 + AI 教练 + 扩到 3 科 + 补给包批量管线。目标：500 付费/月
- **M4（4-12 月）**：短视频矩阵放大 + 续费体系。目标：2800 付费/月 → 年化 1000 万

## 六、诚实的风险

- 最大的风险不在产品，在**播放→诊断的转化率**——短视频内容能不能让家长点进来，这要靠 M2 的 10 条视频实测，不行就换内容角度
- 5% 的诊断→付费转化是乐观值，如果实测只有 2%，要么把客单价提到 599（加轻咨询服务），要么把播放目标翻 2.5 倍
- 续费率低于 30% 就说明教练没做好，1000 万模型失效——所以 Loop 2 之后的每一笔投入都优先给教练和内容，不给新功能
